# Codex Master Shorts Prompt

You are working inside the MrLluminati Gaming Shorts workflow repository.

Read these files first:

- `docs/SHORTS_STRATEGY.md`
- `docs/CONTENT_PILLARS.md`
- `docs/TITLE_AND_HOOK_LIBRARY.md`
- `docs/METADATA_TEMPLATE.md`
- `docs/CODEX_WORKFLOW.md`
- `docs/UPLOAD_CHECKLIST.md`

Task:

Generate YouTube Shorts from the provided raw gameplay footage using the documented strategy.

Rules:

- Prioritize CS2 clips when available.
- Target 10-18 seconds.
- Accept 8-22 seconds only when justified.
- Avoid 30+ second Shorts unless the story is unusually strong.
- Start 0.5-1.5 seconds before action, conflict, or payoff.
- Remove menu screens, dead time, slow walking, waiting, and dragged endings.
- Use emotion-first hooks, not generic or purely technical labels.
- Keep titles accurate to the footage.
- Export 1080x1920 vertical Shorts with H.264/AAC when possible.
- Keep gameplay readable and do not cover crosshair, enemies, kill feed, cars, or main action with text.
- Prefer no external music baked in.
- Create metadata and notes for every Short.
- Generate review frames/contact sheets when possible.
- Do not overwrite old outputs; use a new dated or `v2` batch folder.

Required output per Short:

- final `.mp4`
- metadata `.txt`
- notes `.txt`
- review frames/contact sheet when possible

Use the package format from `templates/metadata_template.txt` and `templates/notes_template.txt`.
