# Codex Workflow

Use this workflow for future MrLluminati Gaming Shorts tasks.

## 1. Read Strategy First

Before editing or generating metadata, read:

- `docs/SHORTS_STRATEGY.md`
- `docs/CONTENT_PILLARS.md`
- `docs/TITLE_AND_HOOK_LIBRARY.md`
- `docs/METADATA_TEMPLATE.md`
- `docs/UPLOAD_CHECKLIST.md`

## 2. Analyze Raw Footage

Scan the source for moments with:

- immediate action
- clear conflict
- visible payoff
- funny or tense audio
- quick betrayal/panic/reaction
- no long context requirement

For each source file, prefer selecting only the strongest 1-3 Short-worthy moments.

## 3. Select The Strongest Moment

Choose based on:

- first-frame strength
- clarity without context
- hook potential
- visible action
- clean ending
- audio usability
- fit with a content pillar

## 4. Cut Strategy

- Start 0.5-1.5 seconds before the action/conflict/payoff.
- Target 10-18 seconds.
- Accept 8-22 seconds when justified.
- Avoid 30+ seconds unless the story is unusually strong.
- Remove dead time, menus, slow walking, inventory, and dragged endings.
- Do not cut mid-dialogue for long-form/story content. For Shorts, avoid partial dialogue unless the line itself is the payoff.

## 5. Export Format

Default Shorts export:

- vertical 9:16
- 1080x1920
- H.264 video
- AAC audio
- readable gameplay center
- no text over crosshair, enemies, kill feed, car, or main action

Prefer no external music baked into the file so YouTube Shorts audio can be added later. Use subtle SFX only when it improves retention.

## 6. Generate Metadata

Create:

- metadata file
- notes file
- optional upload package

Use `templates/metadata_template.txt` and `templates/notes_template.txt`.

## 7. Quality Control

Before final output:

- verify duration
- verify resolution and codec
- create review frames/contact sheet when possible
- check text placement
- check hook accuracy
- check first second
- check ending
- check metadata completeness

## 8. Save Outputs

Use organized output folders:

- `live_video_pipeline/short_form/<batch_name>/`
- `live_video_pipeline/metadata/<batch_name>/`
- `live_video_pipeline/review/<batch_name>/`
- `live_video_pipeline/work/<batch_name>/`

Do not overwrite old outputs. Use `v2`, date suffixes, or clear batch names for replacements.
